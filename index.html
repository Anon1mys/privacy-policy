Privacy Policy

QuRe stores user data locally in the browser using Chrome storage. This includes user-created templates, template collections, and extension settings.

The extension interacts with web page content only to insert user-defined templates into input fields. It does not collect, store, or transmit any content from web pages to external servers.

QuRe does not collect personal data unless the user explicitly chooses to sign in. If the user signs in, authentication is handled via Firebase (Google services). This may include processing the user’s email address and storing basic account information such as user ID and subscription status.

All data processed through Firebase is subject to Google’s Privacy Policy.

The extension does not sell, share, or use user data for advertising or any third-party purposes.

User data is stored locally and can be modified or deleted at any time through the extension settings or by removing the extension.

Permissions such as storage, tabs, and scripting are used strictly to provide core functionality, including saving user settings, detecting the active tab, and inserting templates into supported websites.

No data is transmitted externally except when required for authentication or account-related features explicitly initiated by the user.

This policy may be updated in the future if new features such as cloud synchronization or analytics are introduced.
